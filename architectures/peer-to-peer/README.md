# 对等网络架构 (Peer-to-Peer)

## 什么是 P2P

P2P（对等网络）中每个节点既是客户端又是服务器，没有中心权威节点。节点之间直接通信，数据分散存储在各个节点上。与客户端-服务器的区别：没有专属服务器，所有节点地位平等。

## 核心思想

**节点平等 + 直连通信 + 数据分散**：无中心服务器，每个节点同时提供服务和使用服务。

```
  Node A  ←——→  Node B
    ↕              ↕
  Node D  ←——→  Node C

  每个节点既是 Client 又是 Server
  数据分散存储，查询时全网搜索
  无中心节点，无单点故障
```

关键特征：
- **无中心权威** — 对比客户端-服务器：没有专属服务器节点
- **节点平等** — 每个节点同时是服务提供者和消费者
- **数据分散** — 数据分布在多个节点，不在单一服务器
- **直连通信** — 节点之间直接交互，不需要中转

## 代码示例

运行基础示例：

```bash
python example.py
```

关键代码解读：

1. **PeerNode** — 每个节点同时有 store（服务器角色）和 query（客户端角色）
2. **connect()** — 节点之间直接建立连接，没有中心服务器协调
3. **query()** — 先查本地，没有则向邻居查询，体现既是客户端又是服务器
4. **数据分散** — file1 在 A，file3 在 C，B 通过网络查到 A 的 file1

无中心节点，数据分散存储——这是 P2P 与客户端-服务器的本质区别。

## 进阶示例


进阶示例展示 P2P 的关键能力：**数据复制 + 去中心化共识 + 网络自愈**。

- `decentralized_vote()` — 无中心节点时通过投票达成共识

## 优缺点

**优点**
- 无单点故障——没有中心服务器，任何节点故障不影响整体
- 自扩展——新节点加入即增加存储和计算能力
- 抗审查——无中心控制，难以完全封锁
- 成本低——不需要购买和维护中心服务器

**缺点**
- 数据一致性难——无中心协调，各节点数据可能不一致
- 查询效率低——全网搜索比查中心服务器慢得多
- 安全风险——节点平等意味着恶意节点可以污染数据
- 管理困难——无中心意味着难以统一配置和升级

## 眜实项目中的应用

- **BitTorrent** — 文件分片分散存储，节点互相传输数据片段
- **Bitcoin** — 区块链 P2P 网络，节点共同验证交易和区块
- **IPFS** — 分布式文件系统，内容寻址替代中心化域名
