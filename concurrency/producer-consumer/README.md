# 生产者-消费者模式 (Producer-Consumer Pattern)

## 什么是生产者-消费者模式

生产者-消费者模式通过队列解耦数据的生产与消费。生产者向队列放入数据，消费者从队列取出数据，队列作为缓冲区管理流量控制——生产速度和消费速度不需要匹配。

## 核心思想

**队列解耦生产与消费**：生产者不需要知道谁消费数据，消费者不需要知道谁生产数据。Queue 作为中间缓冲区，既解耦又控制流量。

```
┌──────────┐     ┌─────────┐     ┌──────────┐
│ Producer │ ──→ │  Queue  │ ──→ │ Consumer │
│  放入数据  │     │ 缓冲区   │     │  取出数据  │
│  (快)     │     │ flow控制 │     │  (慢)     │
└──────────┘     └─────────┘     └──────────┘
         ↑                              ↑
     不关心消费者                    不关心生产者
```

关键机制：
- **队列缓冲** — 生产者和消费者通过队列间接交互
- **流量控制** — `maxsize` 限制队列大小，满时阻塞生产者
- **结束信号** — `None` 作为哨兵值通知消费者结束
- **task_done** — 消费者确认处理完成，配合 `join()` 确保所有任务完成

1. **Queue(maxsize=3)** — 队列最大容量为 3，生产者超过此数时会阻塞
2. **producer** — 生成 5 个 item 放入队列，最后放入 `None` 作为结束信号

注意生产者 `time.sleep(0.1)` 比消费者 `time.sleep(0.2)` 快——队列缓冲区正好缓解这种速度差异。

## 优缺点

**优点**
- 解耦——生产者和消费者完全独立，互不干扰
- 流量控制——队列缓冲缓解速度差异
- 可扩展——轻松增加生产者或消费者数量
- 异步处理——生产者不需要等待消费者处理完成

**缺点**
- 队列溢出——生产速度持续快于消费速度时队列可能满
- 延迟——队列缓冲增加了数据处理延迟
- 复杂度——需要管理结束信号和线程同步

## 真实项目中的应用

- **消息队列** — RabbitMQ、Kafka 都是生产者-消费者模式的分布式实现
- **日志系统** — 应用产生日志（生产者），日志处理器消费并写入文件
- **任务调度** — Celery 将任务放入队列，Worker 从队列取出执行
- **流水线** — 多级生产者-消费者串联形成数据处理流水线
