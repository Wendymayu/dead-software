# Human-in-the-Loop (人机协作审批)

## 什么是 Human-in-the-Loop

Human-in-the-Loop 让 Agent 在关键决策点暂停等待人类审批。人类可以批准、拒绝或修改 Agent 的提议动作。这确保高风险决策始终在人类控制下。

## 核心思想

**不是所有决策都该交给AI。低风险操作可以自动化，但高风险操作（删除数据、发送邮件、财务交易）必须有人把关。HITL 是 AI 能力和人类控制之间的平衡点。**

```
HITL 流程：

  Agent 提议动作 ──→ 判断风险等级 ──→ 分流审批策略
                                      │
                    ┌─────────────────┼──────────────────┐
                    │                 │                  │
                低风险             中风险             高风险
                    │                 │                  │
              auto_approve       require_approval        block
                    │                 │                  │
              自动执行          请求人类审批          禁止执行
                    │                 │
                    │          ┌──────┼──────┐
                    │          │      │      │
                    │        批准    拒绝    修改
                    │          │      │      │
                    │       执行   取消   执行修改版
```

关键特征：
- **风险分级** — 根据动作风险等级决定审批策略：低风险自动通过，中风险需审批，高风险禁止
- **人类审批** — Agent 在中风险操作前暂停，等待人类批准、拒绝或修改
- **修改能力** — 人类不只是批准/拒绝，还可以修改 Agent 的提议（如把"发送邮件"改为"发送测试邮件"）
- **安全保障** — 高风险操作永远不能自动执行，必须有人类把关

## 代码示例

运行基础示例：

```bash
python ai-architectures/human-in-the-loop/example.py
```

关键代码解读：

1. **SimulatedHuman** — 模拟人类审批者，根据动作类型自动响应：删除操作被拒绝，查询操作被批准
2. **HITLAgent.propose_and_execute()** — Agent 先提议动作，再请求人类审批，再根据审批结果决定是执行还是取消
3. **审批闭环** — 提议 → 审批 → 执行/取消，Agent 不会绕过人类直接执行高风险操作

核心机制：Agent 不自行决定是否执行，而是将决策权交给人类——AI 提议，人类裁决。

## 进阶示例

运行进阶示例：

```bash
python ai-architectures/human-in-the-loop/advanced.py
```

进阶示例展示三级审批模式与人类修改提议：

1. **auto_approve（低风险）** — 读取配置文件等低风险动作自动通过，无需等待人类，保证效率
2. **require_approval（中风险）** — 发送邮件、更新资料等中风险动作暂停，等待人类审批。人类可以批准、拒绝或**修改提议**——把"发送通知邮件"改为"发送内部测试邮件"，先安全测试再正式上线
3. **block（高风险）** — 删除生产数据库等高风险动作始终禁止执行，无论人类是否同意，策略层面直接阻断

三级审批的核心：不是一刀切地要求所有操作都等人类审批，而是根据风险分级处理——兼顾效率和安全。

## 优缺点

**优点**
- 高风险操作有安全保障——人类把关防止 AI 误操作
- 人类可纠正 AI 错误——审批时发现提议不合理可拒绝或修改
- 满足合规和监管要求——许多行业要求关键决策有人类审计痕迹
- 增强用户信任——用户知道 AI 不会自行执行危险操作

**缺点**
- 人类审批增加延迟——中风险操作必须等待人类响应才能继续
- 依赖人类响应速度——如果审批者不在，Agent 会一直等待
- 审批者可能不负责任地批过一切——"批准疲劳"使审批沦为形式
- 降低自动化效率——即使是简单操作也可能因审批流程变慢

## 真实项目中的应用

- **GitHub Copilot** — 代码建议需要人工确认后才写入文件，不会自动提交
- **AI 医疗诊断** — AI 给出诊断建议，但最终诊断和治疗方案由医生确认
- **金融交易 AI** — 大额交易需风控人员审批，AI 不能自行执行超出限额的交易
- **自动驾驶** — 紧急情况下请求人工介入接管，AI 不独自处理极端场景
