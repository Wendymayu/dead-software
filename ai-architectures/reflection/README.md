# Reflection（反思迭代）

## 什么是 Reflection

Reflection 让 Agent 生成初始输出后自我审视并迭代改进。生成→审视→修改→再审视，直到质量达标或达到最大迭代次数。

## 核心思想

**LLM 既当创作者又当审核者。第一次生成可能不够好，但审视能发现问题，修改能改进。就像写文章——先写初稿，再修改，再润色。**

```
Reflection 循环：

  Question ──→ Generate ──→ Critique ──→ Revise ──→ Critique ──→ ... ──→ Final Answer
                  │             │            │             │
                  │         发现问题        根据问题       再审视修改后
                  │         （太模糊、      修改回答       的版本
                  │          缺例子等）
                  │             │            │             │
              初稿可能不好  审视发现问题  修改能改进   直到审视通过 → 输出
```

关键特征：
- **Generate** — 生成初始回答，可能质量不高但提供了修改基础
- **Critique** — 审视回答，像审核者一样发现问题（太模糊、缺少例子、逻辑漏洞等）
- **Revise** — 根据审视意见逐项修改，每次修改解决一个具体问题
- **收敛终止** — 当审视不再发现问题时，迭代终止，避免无意义的重复修改

## 代码示例

运行基础示例：

```bash
python ai-architectures/reflection/example.py
```

关键代码解读：

1. **CriticAgent** — 审视回答，检查是否模糊、是否缺少例子、是否缺少原因解释，返回具体问题列表
2. **RevisionAgent** — 根据问题列表逐项修改：模糊→补充具体说明，缺例子→添加例子，缺原因→解释机制
3. **ReflectionAgent** — 循环执行 Generate→Critique→Revise：初始回答经过 2-3 次审视和修改，逐步从模糊到完整
4. **收敛检测** — 当 CriticAgent 不再发现问题时，提前终止迭代，返回最终版本

初稿不好没关系，审视发现问题，修改逐步改进——这就是反思的核心机制。

## 进阶示例

运行进阶示例：

```bash
python ai-architectures/reflection/advanced.py
```

进阶示例展示反思的两个关键特性：

1. **质量评分表(Rubric)** — 审视不仅列出问题，还按维度（准确性、完整性、清晰度）逐项评分，每个维度有明确阈值，评分低于阈值才触发修改
2. **收敛终止** — 当所有维度评分都达到阈值时，反思收敛，提前终止迭代。这避免了"永远不满意"的无限循环，也避免简单任务过度迭代浪费时间

评分表让审视从"我觉得有问题"变为"准确性3/4[FAIL]，完整性4/3[PASS]"——有标准、可量化、可收敛。

## 优缺点

**优点**
- 无需外部反馈即可改进——LLM 自身既是创作者又是审核者
- 输出质量可逐步提升——每次修改解决一个具体问题，迭代改进效果明显
- 审视过程可审计——每轮审视发现问题、修改内容都有记录
- 适合需要高质量输出的场景——技术解释、代码生成等对准确性要求高的任务

**缺点**
- 每次迭代都消耗 token——审视和修改各需一次 LLM 调用，成本叠加
- 可能无限循环——没有收敛机制时 Agent 可能永远不满意，需设置最大迭代次数
- 审视质量依赖 LLM 自身能力——LLM 可能审视不出自己的错误（盲点）
- 简单任务过度迭代浪费时间——对已经足够好的回答继续修改是浪费

## 真实项目中的应用

- **Reflexion (Shinn et al., 2023)** — Agent 执行任务后反思失败原因，将反思经验存入记忆指导下次尝试，实现跨任务的口头强化学习
- **Self-Refine (Madaan et al., 2023)** — LLM 对自身输出进行迭代审视和修改，无需外部反馈即可改进文本、代码等多种输出
- **DeepMind AlphaCode** — 代码生成后自我评估测试通过率，不通过的代码触发重新生成，直到通过所有测试
- **OpenAI o1** — 推理模型内部隐式执行 reasoning + reflection，在输出前反复审视推理链的正确性
