# Planning (目标分解)

## 什么是 Planning

Planning 让 Agent 将高层目标分解为可执行的子任务序列，制定计划后逐步执行。如果某步失败，Agent 可以调整计划而非放弃整个目标。与 ReAct 的"边想边做"不同，Planning 是"先想好路线再出发"——先规划全貌，再逐步落地。

## 核心思想

**先规划再执行，失败时局部修订而非全局放弃**：复杂目标无法一步完成，必须先分解为子任务序列，再逐步执行。遇到障碍时修订计划而非从头重来。

```
高层目标: "规划东京旅行"

  [PlanningAgent] ──→ 分解目标 ──→ 生成子任务序列
                                    │
                                    ↓
                    ┌──────────────────────────────────┐
                    │ 1. 研究航班                      │
                    │ 2. 查找酒店                      │
                    │ 3. 查看天气                      │
                    │ 4. 计算预算                      │
                    └──────────────────────────────────┘
                                    │
                          逐步执行 ↓
                                    │
                  ┌─── 步骤失败？ ──→ 修订计划（替换子任务）
                  │                  而非放弃整个目标
                  └─── 步骤成功？ ──→ 继续下一步
```

关键特征：
- **目标分解** — 将模糊的高层目标拆成清晰、可执行的子任务
- **计划驱动** — 先生成完整计划再执行，而非边做边想
- **修订能力** — 某步失败时局部调整计划，而非全局放弃

## 代码示例

运行基础示例：

```bash
python ai-architectures/planning/example.py
```

关键代码解读：

1. **TASK_REGISTRY** — 任务注册表，将子任务名称映射到具体执行函数，实现"计划"与"执行"的解耦
2. **PlanningAgent.plan()** — 接收高层目标，分解为子任务序列（研究航班→查找酒店→查看天气→计算预算）
3. **PlanningAgent.execute()** — 按计划顺序逐步执行每个子任务，通过注册表查找对应函数
4. **计划与执行分离** — Agent 先生成计划再执行，而非边执行边决策——这是 Planning 与 ReAct 的核心区别

Planning 的核心：先规划路线再出发，失败时换路线而非放弃目的地。

## 进阶示例

运行进阶示例：

```bash
python ai-architectures/planning/advanced.py
```

进阶示例展示计划修订与动态调整：

1. **航班太贵 → 修订计划** — 当"研究航班"子任务失败（价格超预算），Agent 将该步替换为"查找替代航班"，从失败位置重新执行而非放弃整个目标
2. **天气恶劣 → 动态调整** — 执行"查看天气"发现暴雨，Agent 修订后续步骤，将酒店调整为室内活动型，利用中间结果影响后续决策

## 优缺点

**优点**
- 可处理复杂多步目标——分解后每步清晰可控
- 计划可审查和调整——人类可在执行前检查计划合理性
- 失败时局部调整而非全局放弃——修订比重新开始更高效
- 适合需要协调多个步骤的任务——任务间有依赖关系时尤为有效

**缺点**
- 计划可能不完整——遗漏关键步骤会导致执行失败
- 过度规划浪费时间——简单任务不需要先规划再执行
- 计划与执行可能脱节——环境变化时原始计划可能失效
- 不适合需要实时反应的场景——计划驱动的延迟高于边做边想

## 真实项目中的应用

- **AutoGPT** — plan-and-execute 模式：Agent 先制定计划再逐步执行，遇障碍时修订
- **LangChain PlanAndExecute** — LangChain 提供的 Planning Agent 实现，将规划器和执行器解耦
- **HuggingGPT** — 用 LLM 规划多模态任务序列（哪个模型处理哪个子任务），再逐步调用 HuggingFace 模型执行
- **Devin** — AI 编程 Agent：plan → execute → verify 三阶段，计划失败时回溯修订
