# Memory（记忆架构）

## 什么是 Memory

Memory 是 Agent 的状态管理架构——短期记忆(对话上下文)、长期记忆(跨会话持久知识)、工作记忆(当前任务状态)。不同类型的记忆有不同的容量、持久性和检索方式。

## 核心思想

**没有 Memory 的 Agent 每次对话都从零开始，像失忆的人。Memory 让 Agent 积累经验、记住偏好、保持连贯。短期记忆保"当前对话"，长期记忆保"所有经验"，工作记忆保"正在做的事"。**

```
三层记忆架构：

  ┌─ ShortTermMemory ──────── 近期对话，容量有限，超限则遗忘
  │   "用户: 我想学Python"
  │   "助手: Python是个好选择"
  │
  │─ LongTermMemory ────────── 持久知识，跨会话存活，按相关性检索
  │   "用户偏好 = 简洁语言"
  │   "Python有GIL限制"
  │
  └─ WorkingMemory ─────────── 当前任务，焦点与进度
      "任务 = 选编程语言"
      "步骤1: 了解偏好 ✓"
```

关键特征：
- **短期记忆(STM)** — 保存当前对话上下文，容量有限，超出时淘汰旧条目
- **长期记忆(LTM)** — 持久保存重要事实和摘要，跨会话存活，支持相关性检索
- **工作记忆(WM)** — 跟踪当前任务的焦点、目标和已完成步骤
- **遗忘与压缩** — STM 按重要性淘汰，对话摘要压缩存入 LTM

## 代码示例

运行基础示例：

```bash
python example.py
```

关键代码解读：

1. **ShortTermMemory** — 保存近期对话，容量限制为5条，超出时丢弃最早的消息，模拟真实 STM 的容量约束
2. **LongTermMemory** — 以字典存储持久事实，通过 key-value 方式存取，跨会话不丢失
3. **WorkingMemory** — 设定当前任务(task)，逐步记录完成的步骤(steps)，追踪任务进度
4. **AgentMemory** — 统一管理三层记忆，对话时写入 STM、识别偏好时存入 LTM、任务信息保存在 WM

三层记忆各司其职：STM 保对话流畅，LTM 保知识积累，WM 保任务聚焦。

## 进阶示例

运行进阶示例：

```bash
python advanced.py
```

进阶示例展示记忆管理的三大策略：

1. **重要性遗忘** — 短期记忆超容量时，不是丢弃最早的消息，而是淘汰重要性评分最低的条目，保留高价值对话
2. **摘要压缩** — MemorySummarizer 将多条对话压缩为一条摘要存入长期记忆，从 N 条对话变为 1 条摘要，节省存储空间
3. **相关性检索** — 长期记忆不再按精确 key 查找，而是用关键词重叠评分检索最相关的事实，支持模糊匹配

遗忘不是简单的"丢弃旧的"，而是"保留重要的"；检索不是精确查找，而是相关性排序。

## 优缺点

**优点**
- Agent 可积累知识跨会话使用——长期记忆让每次对话都不是从零开始
- 保持对话连贯——短期记忆提供上下文，避免 Agent 重复或遗忘
- 记住用户偏好——长期记忆存储用户习惯，提供个性化交互
- 从经验中学习——摘要压缩将历史对话提炼为可复用的知识

**缺点**
- 记忆容量有限必须取舍——遗忘机制可能丢失有用信息
- 长期记忆检索可能不精确——相关性匹配无法保证100%准确
- 记忆冲突——旧记忆可能与新事实矛盾（用户偏好改变后旧偏好仍在）
- 存储和检索成本——长期记忆规模增大后检索开销上升

## 真实项目中的应用

- **MemGPT/Letta** — 将 LLM 视为操作系统，Memory 作为虚拟内存管理，自动在主存(STM)和磁盘(LTM)间调度
- **LangChain memory 模块** — 提供多种记忆类型（Buffer、Summary、Vector），支持对话历史管理和持久化
- **Claude conversation memory** — Anthropic 的对话记忆机制，保持多轮对话上下文连贯
- **ChatGPT memory feature** — OpenAI 的记忆功能，跨对话记住用户偏好和事实
- **Zep** — 开源记忆服务，为 LLM 应用提供长期记忆管理和语义检索
